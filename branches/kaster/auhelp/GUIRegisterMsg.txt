
GUIRegisterMsg

Регистрация пользовательских функций для известных ID Windows сообщений (WM_MSG).

GUIRegisterMsg ( msgID, "function" )

---  Таблица параметров  ---
msgID
	ID Windows сообщение (see Appendix: Windows Message Codes).
function
	Имя пользовательской функции вызываемой, когда появляется сообщение или пустая строка "" для освобождения функции от сообщения.

---  Таблица возвращаемых значений  ---
Успех:	1
Ошибка:	0

---  Примечания  ---
!!! Чтобы функция была работоспособной, нужно определить максимум 4 параметра функции иначе функция не будет вызываться !!!
То есть:
Func MyUserFunction($hWndGUI, $MsgID, $WParam, $LParam)
...
EndFunc

Или

Func MyUserFunction($hWndGUI, $MsgID)
...
EndFunc

Когда функция вызывается, то 4 параметра принимают следующие значения:
---  Таблица  ---
Позиция	Параметр	Описание
1	hWnd	Дескриптор окна GUI, который получил сообщение.
2	Msg	ID Windows сообщения.
3	wParam	Первый параметр сообщения как шестнадцатеричное значение.
4	lParam	Второй параметр сообщения как шестнадцатеричное значение.

До 256 пользовательских функций могут быть зарегистрированы на ID сообщения.

По умолчанию после окончания функции выполнятся внутренний обработчик сообщения AutoIt .
Этого не произойдёт, если ваш Return имеет значение (Смотрите WM_COMMAND в примере) или, если вы используете 'Return' без какого либо значения.
Используя Return без любого возвращаемого параметра внутренний обработчик сообщения AutoIt (если он есть для этого сообщение) не будет выполнятся!

!!! Если вы хотите, чтобы AutoIt выполнил внутренний обработчик для сообщения, возвратите переменную $GUI_RUNDEFMSG (в GUIConstantsEx.au3) из функции (смотрите также примеры) !!!
То есть если вы хотите выполнить Return раньше, чем функция закончится и также выполнить внутренний обработчик сообщения AutoIt.

Внимание: блокировка выполняющейся пользовательской функции, например командой "Msgbox()" может привести к непредсказуемому поведению, вернуть управление системе нужно как можно быстрее !!!

Некоторые элементы управления поглощают внутренние специфические ID Windows сообщения, поэтому их регистрация не будет иметь эффекта, например: WM_CHAR, WM_KEYDOWN, WM_KEYUP поглощаются элементами Edit.
