
GUICtrlSetData

Изменение параметров элемента.

GUICtrlSetData ( controlID, data [, default] )

---  Таблица параметров  ---
controlID
	Идентификатор (controlID) элемента возвращаемый функцией GUICtrlCreate....
data
	Для Combo, List, ListView, ListViewItem: устанавливает список пунктов с разделителем Opt("GUIDataSeparatorChar",...).
	Для Progress: устанавливает проценты.
	Для Slider: устанавливает величину.
	Для Group, Label, Button, Checkbox, Radio, Combo, List, Input, Edit, TabItem, TreeViewItem: заменяет текст.
	Для Date : устанавливает дату или время в зависимости от стиля элемента.
	Для Dummy: устанавливает величину.
default
	[опционально] Для Combo, List: значение по умолчанию.
	Для Edit, Input: если не пустая строка (""), то вставляется в текущую позицию (где установлен курсор).

---  Таблица возвращаемых значений  ---
Успех:	Возвращает 1.
Ошибка:	Возвращает 0.
	Возвращает -1 в случае неверных данных

---  Примечания  ---
Для элементов Combo или List :
Если "данные" уже содержатся в элементе, то эти данные становятся текущими по умолчанию.
Если "данные" начинаются с разделителя (GUIDataSeparatorChar) или пустой строки ("") то предыдущий список удаляется.
Если "данные" не содержатся в элементе, но требуется добавить и сделать текущими, то используйте GUICtrlSetData ( $controlID, "данные" , "данные" )

Для элементов ListView, ListViewItem :
Для обновления определённой колонки просто забыть о других т.е. "| | Обновление" для обновления 3-ей колонки.
Если обновление выполняется пустотой, то колонка / подпункт будет стёрт. Для примера "|" очистит вторую колонку / подпункт.

Для элементов Date, Monthcal :
Формат "данных" даты -  "yyyy/mm/dd".

Для элементов Edit :
При вставке "данных" новые данные заменяет предыдущие. Чтобы "данные" дописывались в конец предыдущих данных, добавьте в параметр default любое значение, например 1.
