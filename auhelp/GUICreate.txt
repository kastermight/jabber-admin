
GUICreate

Создаёт GUI окно.

GUICreate ( "title" [, width [, height [, left [, top [, style [, exStyle [, parent]]]]]]] )

---  Таблица параметров  ---
title
	Заголовок окна.
width
	[опционально] ширина окна.
height
	[опционально] высота окна.
left
	[опционально] координата левого края. По умолчанию (-1), окно создаётся в центре. Если параметр задан, нужно также задать координату верхнего края.
top
	[опционально] координата верхнего края. По умолчанию (-1), в центре.
style
	[опционально] определяет стиль окна. Смотрите Стили элементов GUI в приложении.
	Используйте -1 для стандартного стиля, который включает стили $WS_MINIMIZEBOX, $WS_CAPTION, $WS_POPUP и $WS_SYSMENU.
	Некоторые стили всегда включены: $WS_CLIPSIBLINGS и $WS_SYSMENU, если добавлены $WS_MAXIMIZEBOX или $WS_SIZEBOX.
exStyle
	[опционально] расширенные стили окна. Смотрите таблицу расширенных стилей ниже. По умолчанию -1.
parent
	[опционально] дескриптор предыдущего окна, для которого текущее окно становится дочерним.

---  Таблица возвращаемых значений  ---
Успех:	Возвращает дескриптор окна.
Ошибка:	Возвращает 0, если окно не может быть создано, и устанавливает @error равным 1.

---  Примечания  ---
По умолчанию окно не способно изменить размер и не может быть развёрнуто. Поэтому WS_SIZEBOX или WS_MAXIMIZEBOX могут быть использованы в параметрах стиля.
При добавлении стиля не забывайте объединять его со стандартным, т.е. просто добавление стиля WS_SIZEBOX не установит стили WS_MINIMIZEBOX, WS_CAPTION, WS_POPUP и WS_SYSMENU. Так что лучший способ создания окна с изменением размеров является WS_OVERLAPPEDWINDOW.
При использовании $WS_EX_MDICHILD позиция указывается относительно клиентской области родительского окна. C $WS_EX_LAYERED возможно иметь прозрачный пиксель на фоне пикселов окна.
Добавление стиля $WS_CLIPCHILDREN позволяет избежать мерцания при изменении размера окна, например, содержащего элемент Edit.
Можно разрешить перетаскивание окна за клиентскую область (например, если отключён стиль $WS_CAPTION), используя $WS_EX_CONTROLPARENT в дополнительных параметрах стиля.

Чтобы объединить стили со стилем по умолчанию используйте BitOr($GUI_SS_DEFAULT_GUI, новый стиль,...).
Размер определяется размером клиентской области окна. Кайма и заголовок сделают окно чуть больше, чем указано. Использование элементов главного меню также изменяют высоту окна.

Дополнительная таблица стилей
---  Таблица  ---
Дополнительный стиль	результат
$WS_EX_ACCEPTFILES	Разрешает элементам Edit или Input в пределах окна получать путь / имена методом перетаскивания в окно (drag and drop). Также элементы управления должны иметь состояние $GUI_DROPACCEPTED, устанавливаемое функцией GUICtrlSetState(). Для других элементов метод перетаскивания (drag&drop) осуществляется макросами @GUI_DRAGID, @GUI_DRAGFILE, @GUIDROPID.
$WS_EX_APPWINDOW	Заставляет дочернее окно отображаться на панели задач.
$WS_EX_CLIENTEDGE	Края имеющие вид утопленного окна.
$WS_EX_CONTEXTHELP	Добавляет системную кнопку - знак вопроса в заголовок окна. Несовместим с WS_MAXIMIZEBOX или WS_MINIMIZEBOX.
$WS_EX_DLGMODALFRAME	Создаёт окно с двойной каймой; при необходимости окно может быть создано с заголовком при указании стиля WS_CAPTION.
$WS_EX_MDICHILD	Создать дочернее окно (имитация MDI).
$WS_EX_OVERLAPPEDWINDOW	Объединяет стили WS_EX_CLIENTEDGE и WS_EX_WINDOWEDGE.
$WS_EX_STATICEDGE	Создать окно с границами в трех-мерном стиле, используется для управляющих элементов не предназначенных для ввода данных от пользователя.
$WS_EX_TOPMOST	Поверх всех окон, даже если неактивно.
$WS_EX_TRANSPARENT	Прозрачное окно (подложка из окон уже отрисована).
$WS_EX_TOOLWINDOW	Создаёт сервисное окно, т.е. окно предназначенное для использования как плавающая панель инструментов. Ширина заголовка окна меньше обычного и уменьшенный шрифт заголовка. Окно не появляется в панели задач и в списке выбора при нажатии ALT + TAB. Если окно инструментов имеет системное меню, его значок не отображается в строке заголовка. Тем не менее, окно отображает системное меню по нажатию ALT+SPACE.
$WS_EX_WINDOWEDGE	Окно с приподнятыми краями.
$WS_EX_LAYERED	Создаёт многоуровневое окно. Но оно не может быть использовано в качестве дочернего окна.

Чтобы использовать переменные, указанные выше, вы должны  добавить #include <WindowsConstants.au3> в ваш скрипт.

Внимание: дескриптор, возвращаемый этой функцией является реальным дескриптором окна, точно такой же, как и при получении дескриптора функцией WinGetHandle.
