
StringFormat

Возвращает форматированную строку (имитация функции printf() из языка C).

StringFormat ( "format control", var1 [, ... var32] )

---  Таблица параметров  ---
format control
	Формат и флаги для использования (см. примечания).
var1...var32
	До 32 переменных, которые будут выводиться в соответствии с "format control".

---  Таблица возвращаемых значений  ---
Возвращает форматированную строку с "variable format" (изменяемым форматом), заданным в соответствии с параметрами "format control".

---  Примечания  ---
Для предотвращения переполнения буфера, размер каждой "переменной" (varXX) ограничен 65535 символами.
Экранированные символы, такие как \n (@LF), \r (@CR), \t (@TAB), могут содержаться в "format control". Однако, если нужно использовать символ "\", то его следует записывать, как \\. То же самое и для "%" - %%.

"variable format" is; %[flags] [width] [.precision] type

		Спецификация width (ширины)

Вторым опциональным полем в спецификации формата является аргумент width. width - неотрицательное десятичное целое число, контролирующее минимальное число выводимых символов. Если количество символов меньше указанного аргумента width, то добавляются пробелы слева или справа, в зависимости от указанного флага выравнивания. Если аргумент width начинается с 0, то нули добавляются до минимальной ширины .

Ширина не обрезает выводимое значение. Если количество выводимых символов больше указанного аргумента width или width не задан, то выводятся все символы.

		Спецификация type (типа)

---  Таблица  ---
Тип	Тип переменной	Выходной формат
d, i	Целое	Десятичное целое число.
o	Целое	Восьмеричное целое число без знака.
u	Целое	Десятичное целое число без знака.
x	Целое	Шестнадцатеричное целое число без знака, используются "abcdef".
X	Целое	Шестнадцатеричное целое число без знака, используются "ABCDEF".
e	С плавающей точкой	Запись имеющая вид [ - ]d.dddd e [знак]ddd, где d единственная десятичная цифра, dddd одна и более десятичных цифр, ddd точно три десятичных цифры и знак + или -.
E	С плавающей точкой	Идентично e формату, кроме этого E, в отличии от e, выводит показатель.
f	С плавающей точкой	Запись величины имеет вид [ - ]dddd.dddd, где dddd одна и более десятичных цифр. Количество цифр перед десятичной точки зависит от величины числа, и количество цифр после десятичной точки зависит от запрошенной точности.
g	С плавающей точкой	Запись величины выводится в f или e формате, в зависимости от того, в каком виде запись является более компактной для заданного значения и точности.  Формат e используется только тогда, когда показатель величины меньше -4 или больше или равен точности аргумента. Завершающие нули отбрасываются, а точка появляется, если за ней будут следовать цифры.
G	С плавающей точкой	Идентично g формату, кроме этого E,в отличии от e, выводит показатель (если необходимо).
s	Строка	Строка.

		Спецификация flags (флагов)

---  Таблица  ---
Флаг	Значение	По умолчанию
-	Левое выравнивание результата в пределах ширины области.	Правое выравнивание.
+	Префикс выходной величины, знак (+ или -) если величина имеет тип записи.	Знак появляется только для отрицательных величин (-).
0	Если width с префиксом 0, нули добавляются до заполнения минимальной ширины. Если 0 и появился, то 0 игнорируется. Если 0 является в формате целого числа (i, u, x, X, o, d) то 0 игнорируется.	Отсутствует заполнение.
Blank	Префикс величины отсутствует, если выходная величина положительна; blank игнорируется если оба blank и + флаги появляются.	Отсутствует blank.
#	Когда используется с o, x, или X формат, то # флаг приписывается любая ненулевая выходная величина с 0, 0x, или 0X, соответственно.	Отсутствует blank.
#	Когда используется с e, E, или f формат, то # флаг форсирует выходную величину содержать десятичную точку во всех случаях.	Десятичная точка появляется только если цифры следуют за ней.
#	Когда используется с g или G формат, то #  флаг форсирует выходную величину содержать десятичную точку во всех случаях и предотвращает усечение конечных нулей.Игнорируется, когда используется с c, d, i, u, или s.	Десятичная точка появляется только если цифры следуют за ней. Конечные нули исключены.

		Спецификация .precision (точности)

Третьим параметром области формата спецификации является спецификация точности. Это определённое целое число, предшествующее точке (.), которое определяет количество символов вывода, количество десятичных порядков, или количество значимых цифр (см. таблицу ниже). В отличие от ширины спецификации, спецификации точности может привести либо усечению выходной величины или округлению с плавающей запятой. Если точность определена как 0 и значение для преобразования равно 0, то результат вывода не имеет символов, как показано ниже:

StringFormat( "%.0d", 0 );      /* Никакие символы не будут возвращены */

	Как на точность влияет тип

Тип	Значение	По умолчанию

e, E	Точность определяет количество цифр для вывода после десятичной точки. Последняя цифра вывода округляется.	По умолчанию точность - 6; если точность 0 или точка (.) появляется без числа, следующего за ним, десятичная точка не выводится.  
f 	Точность величины определяет количество цифр после запятой. Десятичная точка появляется только если цифры следуют за ней. Величина округляется до соответствующего количества цифр	По умолчанию точность - 6; если точность 0 или точка (.) появляется без числа, следующего за ним, десятичная точка не выводится. 
g, G	Точность определяется максимальным количеством значащих цифр вывода.	Шесть значащих цифр выводятся, любые конечные нули усечены. 
s	Точность определяется максимальным количеством символов для вывода. Символы сверх точности не выводятся.	Символы выводятся до первого нулевого символа.
